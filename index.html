Comparing Q-Learning and Policy Gradient with RBF Features on MountainCar

A study of value-based vs policy-based reinforcement learning

Hesam Mohebi · University of Genova

Motivation

Reinforcement learning provides a principled framework for sequential decision making under uncertainty. An agent interacts with an environment, receiving rewards and learning to maximize long-term return.

This project compares two major families of RL methods:

  • Value-based methods — learn the value of state-action pairs, derive a policy indirectly
  • Policy-based methods — learn the policy directly from experience

MountainCar is a classic benchmark: simple enough to analyze clearly, yet challenging enough to require sophisticated exploration and long-term planning.

With a continuous state space, function approximation is essential — making this comparison both practical and theoretically informative.

The MountainCar Problem

A car is placed in a valley between two hills. The goal is to reach the top of the right hill.

The engine is too weak to drive directly uphill — the car must build momentum by rocking back and forth.

  • State: position \(x \in [-1.2, 0.6]\), velocity \(\dot{x} \in [-0.07, 0.07]\)
  • Actions: push left (0), no push (1), push right (2)
  • Reward: \(-1\) per time step until goal is reached

The environment terminates when the car reaches position \(\geq 0.5\) (success) or after 200 steps (failure).

Q-Learning

Q-learning is a value-based method that estimates the action-value function \(Q(s,a)\) — the expected return from taking action \(a\) in state \(s\) and following the optimal policy thereafter.

The policy is obtained by selecting the action with the highest estimated Q-value:

$$\pi(s) = \arg\max_a Q(s,a)$$

The core update rule (semi-gradient with function approximation):

$$\mathbf{w} \leftarrow \mathbf{w} + \alpha \left[ r + \gamma \max_{a'} Q(s',a') - Q(s,a) \right] \nabla_{\mathbf{w}} Q(s,a)$$

Policy Gradient / Actor-Critic

Policy gradient methods learn the policy directly by parameterizing \(\pi_\theta(a|s)\) and optimizing expected return via gradient ascent.

The Actor-Critic architecture combines:

  • Actor: softmax policy that selects actions
  • Critic: estimates state value \(V(s)\) as a baseline

The advantage \(\delta = r + \gamma V(s') - V(s)\) tells the actor whether the action was better or worse than expected, reducing variance.

$$\theta \leftarrow \theta + \alpha_\theta \, \delta \, \nabla_\theta \ln \pi_\theta(a|s)$$
$$\mathbf{w} \leftarrow \mathbf{w} + \alpha_w \, \delta \, \nabla_{\mathbf{w}} V_{\mathbf{w}}(s)$$

Why Function Approximation?

MountainCar has a continuous state space: position and velocity are real-valued, creating an infinite number of possible states.

Tabular methods would require discretizing the space into bins — losing resolution, introducing aliasing, and scaling poorly.

Function approximation enables:

  • Generalization across similar states
  • Compact representation of value/policy
  • Efficient learning from limited experience

Both Q-learning and Actor-Critic in this project use the same RBF feature representation, ensuring a fair comparison.

RBF Feature Map

Both algorithms share a common feature representation based on Radial Basis Functions (RBFs):

  1. Normalize state dimensions to \([0,1]\)
  2. Place a grid of centers across the normalized space
  3. Compute Gaussian activations:
$$\phi_i(s) = \exp\!\left( -\frac{\|s - c_i\|^2}{2\sigma^2} \right)$$

Nearby centers activate strongly; distant ones contribute little. This creates a smooth, local feature representation.

Q-Learning — Implementation Details

Q-values are approximated as a linear combination of RBF features:

$$Q(s,a) = \mathbf{w}_a^\top \boldsymbol{\phi}(s)$$

Each action has its own weight vector. The agent uses \(\varepsilon\)-greedy exploration during training — choosing a random action with probability \(\varepsilon\), and the greedy action otherwise.

The semi-gradient update adjusts weights toward the TD target, but note the deadly triad: combining bootstrapping, off-policy learning, and function approximation can cause instability.

Actor-Critic — Implementation Details

The actor uses a softmax policy over RBF features:

$$\pi_\theta(a|s) = \frac{\exp(\theta_a^\top \phi(s))}{\sum_{a'} \exp(\theta_{a'}^\top \phi(s))}$$

The critic estimates the state value as:

$$V_{\mathbf{w}}(s) = \mathbf{w}^\top \boldsymbol{\phi}(s)$$

The TD error \(\delta = r + \gamma V(s') - V(s)\) serves as the advantage estimate. The critic reduces variance of the policy gradient, leading to more stable learning.

Interpretation

Both methods successfully learn to solve MountainCar, but exhibit different trade-offs:

  • Q-Learning may achieve slightly better final performance after full convergence
  • Actor-Critic can learn competitively in early training phases
  • Results depend heavily on the quality of the feature representation and hyperparameter choices

Conclusion

  • MountainCar requires long-term planning due to sparse reward and momentum-based dynamics
  • RBF features provide a shared, fair representation for comparing methods in continuous state space
  • Both Q-Learning and Actor-Critic solve the task with proper tuning
  • The two families offer different strengths — value-based precision vs policy-based flexibility
